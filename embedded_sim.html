Overview

This project is a task scheduling system designed to optimize the allocation of tasks across different hardware instances. The system supports various scheduling rules, conditional tasks, and unpredicted tasks. It also provides functionality for simulating task execution, calculating costs, and generating schedules.

Project Structure

The project is organized into several key components, each responsible for different aspects of the system:

Cost_List Class

The Cost_List class is the core of the project. It manages tasks, hardware instances, and scheduling rules. Key functionalities include:

  • Task Distribution: Implements various scheduling rules for task allocation.
  • Instance Management: Handles the creation and management of hardware instances.
  • Task Execution: Simulates task execution and calculates execution times and costs.
  • Printing and Reporting: Provides methods to print schedules, costs, and other relevant information.

Hardware and Instance Management

The project includes classes for managing hardware resources (Hardware) and task instances (Instance). These classes are used to represent the physical hardware and the tasks assigned to them.

Task Graph

The Graf class represents the task graph, which defines the dependencies between tasks. It provides methods for traversing the graph (e.g., BFS, DFS) and managing task relationships.

Times and Costs

The Times class manages the time and cost associated with executing tasks on different hardware. It provides methods for retrieving and normalizing these values.

Key Functionality

Task Scheduling

The system supports multiple scheduling rules, which are implemented in the taskDistribution method. These rules determine how tasks are allocated to hardware instances based on criteria such as time, cost, and hardware availability.

Conditional Tasks

Conditional tasks are tasks that are executed only if certain conditions are met. The system evaluates these conditions during task execution and skips tasks if the conditions are not satisfied.

Unpredicted Tasks

Unpredicted tasks are tasks that are not known at the time of scheduling. The system handles these tasks dynamically by creating new instances or reallocating existing resources.

Simulation and Reporting

The system simulates task execution and provides detailed reports on task schedules, execution times, and costs. The printSchedule and printInstances methods are used to generate these reports.

Usage

To use the system, follow these steps:

  1. Load the configuration and task data using the Load_Config and Load_From_File methods.
  2. Define the task graph and hardware resources.
  3. Choose a scheduling rule and call the taskDistribution method to allocate tasks.
  4. Simulate task execution using the runTasks method.
  5. Generate reports using the printSchedule, printInstances, and other printing methods.
